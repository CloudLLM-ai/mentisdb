Backup and Restore

MentisDB 0.9.2 adds backup and restore commands to the mentisdb CLI. You can now create compressed .mbak archives of your entire MENTISDB_DIR — all chains, agents, skills, and optionally TLS certificates — and restore them on any machine.

What It Is

A backup is a single .mbak ZIP archive named mentisdb-YYYY-MM-DD-HH-MM-SS.mbak containing:

  • mentisdb.manifest.json — metadata and SHA-256 checksums for every file
  • All chain data files (*.tcbin, *.agents.json, *.entity-types.json, vector sidecars)
  • mentisdb-registry.json — global chain registry
  • mentisdb-skills.bin — skill registry (if present)
  • mentisdb-webhooks.json — webhook registrations (if present)
  • tls/ — TLS certificates and keys (opt-in)

The manifest uses SHA-256 so restore can verify every file before writing anything to disk. If any check fails, the restore aborts with an error and the target directory is left untouched.

How It Works

Consistency Model on Backup

When you run mentisdb backup, the CLI checks whether the daemon is running at http://127.0.0.1:9472. If it is, it calls POST /v1/admin/flush to durably flush all open chains before reading any files. This ensures the backup captures a consistent on-disk state even if the daemon has AUTO_FLUSH=false.

Daemon running: all chains are flushed first → files are read in a consistent state.

Daemon stopped: files are captured as-is. Nothing is writing, so the on-disk state is already consistent.

Use --flush to force a flush via the REST API regardless of the automatic detection. Pass --include-tls to include TLS certificates and keys (these are excluded by default since they are sensitive).

Restore Behavior

Restore is idempotent by default: if a file already exists in the target directory, it is skipped. This means you can safely restore a backup into an existing MENTISDB_DIR without overwriting newer data.

If any files in the archive already exist in the target directory and --overwrite is not passed, the CLI shows an interactive prompt listing the conflicting files and asks for confirmation before proceeding.

Interactive prompt: when files exist and --overwrite is not passed, the CLI asks: "overwrite existing files? [y/N]".

Pass --overwrite to skip the prompt and replace all files with their backed-up versions unconditionally.

CLI Commands

Create a backup

# Default: platform MENTISDB_DIR → ./mentisdb-YYYY-MM-DD-HH-MM-SS.mbak
mentisdb backup

# Custom output path
mentisdb backup --output /backups/mentisdb-2026-04-14.mbak

# Include TLS certificates
mentisdb backup --include-tls

# Explicit flush (automatic if daemon is running)
mentisdb backup --flush

# Full example
mentisdb backup --dir ~/.cloudllm/mentisdb --output /backups/mentisdb-2026-04-14.mbak --include-tls

Restore a backup

# Idempotent restore (preserves existing files)
mentisdb restore /backups/mentisdb-2026-04-14.mbak

# Idempotent restore to a specific directory
mentisdb restore /backups/mentisdb-2026-04-14.mbak --dir ~/.cloudllm/mentisdb

# Force overwrite of all files
mentisdb restore /backups/mentisdb-2026-04-14.mbak --overwrite

REST API Endpoint

The flush endpoint is also available as a standalone REST endpoint for programmatic use: 

POST http://127.0.0.1:9472/v1/admin/flush

Returns {"status": "flushed"} on success. The endpoint iterates over all open chains and calls flush() on each BinaryStorageAdapter, blocking until all writes are durable.

Security note: the /v1/admin/* endpoints are bound to the loopback address (127.0.0.1) by default. They are not exposed externally. The flush endpoint has no authentication because it is a local-only admin operation.

Archive Format

FileRequiredDescription
mentisdb.manifest.jsonYesSHA-256 manifest with format version and timestamps
mentisdb-registry.jsonYesGlobal chain registry
*.tcbinYesBinary chain storage files
*.agents.jsonYesPer-chain agent registries
*.entity-types.jsonYesPer-chain entity type definitions
*.vectors.*.jsonYesVector search sidecar files
mentisdb-skills.binNoSkill registry (if present)
mentisdb-webhooks.jsonNoWebhook registrations (if present)
tls/*NoTLS certs and keys (only if --include-tls was used)

Security Notes

  • TLS certificates are excluded by default. Pass --include-tls only when you need to back up certificates. Be careful about where you store the resulting archive.
  • The REST flush endpoint is loopback-only. It cannot be reached from external hosts. There is no authentication on it because it is intended as a local admin operation.
  • Restore verifies before writing. Every file's SHA-256 is checked against the manifest before any bytes are written to disk. A corrupted archive will never partially overwrite your data.
