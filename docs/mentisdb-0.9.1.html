MentisDB 0.9.1 — LLM Extraction, Federated Search, pymentisdb, and Brew-First Setup

0.9.1 is the biggest release in MentisDB history — four major features land together, plus the first officially published Python client. This post covers all of them.

LLM-Extracted Memories

The most-requested feature is here. Convert raw agent text, conversation logs, or terminal transcripts into structured ThoughtInput records — ready to review and append to your memory chain — using any OpenAI-compatible endpoint.

MentisDB calls GPT-4o (or your preferred model) with a structured extraction prompt and receives back a JSON array of memory records. You review them, optionally sign them with an Ed25519 key, then append them to your chain.

How It Works

The llm-extraction feature is enabled by default. Configure with three environment variables: 

export OPENAI_API_KEY="sk-..."           # required
export LLM_BASE_URL="https://api.openai.com/v1"  # defaults to OpenAI
export LLM_MODEL="gpt-4o"                # defaults to gpt-4o

Then call extract_memories_from_text() from Rust: 

use mentisdb::{LlmExtractionConfig, extract_memories_from_text};

let config = LlmExtractionConfig::from_env()?;
let result = extract_memories_from_text(raw_text, &config, None).await?;

// Review before appending
for thought in &result.thoughts {
    println!("{:?}", thought);
}

println!("Tokens used: {:?}", result.usage);

ExtractionResult

The returned ExtractionResult contains:

  • thoughts: Vec<ThoughtInput> — the extracted memory records, each with thought_type, content, importance, confidence, tags, concepts, relations
  • model: String — the model identifier that produced the extraction
  • usage: TokenUsage — prompt_tokens, completion_tokens, total_tokens
Security note: LLM output is untrusted. Always review and optionally sign extracted thoughts before appending to a chain.

Under the Hood: openai-rust2

The feature uses openai-rust2 — the same HTTP client powering cloudllm — with:

  • Connection pooling (reused across extractions)
  • Automatic 3× retry on 429 Too Many Requests and 5xx errors
  • 60-second timeout
  • Temperature 0.1 for deterministic extraction
  • JSON object response format (ResponseFormat::JsonObject)

Federated Cross-Chain Search

Specialist agents can now branch from a shared base chain and still search across their entire lineage. ancestor_chain_keys() walks BranchesFrom relations to discover parent chains; ranked search on a branch automatically includes ancestor results annotated with chain_key.

This enables a fleet memory hierarchy pattern:

  • Shared base chain with company-wide knowledge
  • Per-project specialist chains that diverge via mentisdb_branch_from
  • Queries on a specialist chain surface both specialist and base-chain memories
// Branch from shared base at a checkpoint thought
POST /v1/chains/branch
{ "source_chain_key": "company-memory", "branch_thought_id": "uuid-of-checkpoint" }

// Ranked search on "project-x" now includes company-memory results
POST /v1/ranked-search
{ "text": "how do we handle billing disputes?", "chain_key": "project-x" }

pymentisdb — Official Python Client

pymentisdb is now live on PyPI. The Python client covers the complete MentisDB REST API surface with full type hints and a LangChain-compatible memory interface. 

pip install pymentisdb
# with LangChain integration:
pip install pymentisdb[langchain]

Basic Usage

from pymentisdb import MentisDbClient, ThoughtType

client = MentisDbClient()  # defaults to http://127.0.0.1:9472

# Append a thought
thought = client.append_thought(
    thought_type=ThoughtType.INSIGHT,
    content="Rate limiting is the real bottleneck."
)
print(f"Appended: {thought.id}")

# Semantic search
results = client.ranked_search(text="performance optimization")
for hit in results.results:
    print(f"[{hit.score.total:.3f}] {hit.thought.content}")

# Context bundles
bundles = client.context_bundles(text="cache invalidation", limit=5)

LangChain Integration

from pymentisdb import MentisDbMemory
from langchain_openai import ChatOpenAI

memory = MentisDbMemory(
    chain_key="my-project",
    agent_id="assistant"
)

llm = ChatOpenAI(model="gpt-4")
chain = llm.with_memory(memory)

Wizard Brew-First Install

The interactive setup wizard now tries brew install mcp-remote before npm. If Homebrew mcp-remote is available, it detects it via shebang inspection — Homebrew installs a proper shebang pointing to the correct Node version, so MentisDB writes the minimal config using mcp-remote directly as the command (no Node.js wrapper needed).

This matters because mcp-remote requires Node ≥ 20 (it uses the File global added in Node 20). Homebrew-managed Node is always the right version; npm-installed mcp-remote may resolve an older Node via #!/usr/bin/env node, so the wizard wraps it with an explicit node path.
